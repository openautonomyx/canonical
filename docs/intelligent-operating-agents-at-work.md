# Intelligent Operating Agents At Work

Intelligent operating agents are not chatbots, prompts, workflows, or tool wrappers.

They are contract-bound identities that observe, reason, and behave to produce outcomes.

## Canonical Frame

```text
Identity
    ↓
Contract
    ↓
ORB
    ↓
Outcome
```

## ORB

```text
Observe
    ↓
Reason
    ↓
Behave
```

An intelligent operating agent works by repeatedly moving through this loop.

### Observe

The agent observes the current state of work.

It gathers context, evidence, signals, memory, events, constraints, and open commitments.

### Reason

The agent reasons within a contract.

It evaluates objectives, alternatives, policies, risk, trust, approvals, and expected outcomes.

### Behave

The agent behaves by taking accountable action.

It performs work, calls tools, updates systems, publishes artifacts, delegates tasks, escalates to humans, or changes state.

## At Work

At work means the agent is not isolated.

It operates with and around edge participants:

- humans
- organizations
- agents
- systems
- communities

The human remains at the edge, not at the center.

The center is stable:

```text
Identity
Contract
ORB
Outcome
```

## Operating Rule

No agent acts without identity.
No identity operates without contract.
No contract executes without ORB.
No ORB cycle is complete without outcome.
No outcome is trusted without provenance.

## Trust Boundary

Trust is accumulated from observed behavior over time.

Governance constrains what may happen.
Provenance records what did happen.
Lifecycle determines whether the identity may continue operating.
